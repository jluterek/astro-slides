# Built

One file per completed phase. Each file is the **distilled summary** of what shipped, written when the phase closed.

A distilled doc should answer, in roughly one screen:

- What did this phase build?
- How is it organized in the codebase (entry points, key files)?
- What were the meaningful decisions and trade-offs?
- What surprised us during the work?
- What's still loose or worth knowing for future work?

The detailed history lives in `todo/archive/NN-phase-name/`. Read that only when this distilled summary isn't enough.

See `todo/README.md` § *Completing a phase* for the workflow.
