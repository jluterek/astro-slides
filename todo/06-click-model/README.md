## Goal

Implement parse-time click resolution per ADR-0008. Each `<Click>`, `<After>`, `<Clicks>` invocation is identified by a remark plugin during MDX compile, assigned a deterministic step index, and the per-slide total is emitted as static metadata. The runtime is purely a consumer of the pre-computed plan.

## Exit criteria

- [ ] `<Click at="â€¦">`, `<After>`, `<Clicks>` components in `packages/client/` implemented.
- [ ] Remark/rehype plugin walks the slide MDX AST, finds click components, assigns step indices, and writes the per-slide step list and total to the slide's frontmatter sidecar module.
- [ ] Relative offsets (`at="+1"`, `at="+0"` = `<After>`) and ranges (`at="[2,5]"`) supported and resolved at parse time.
- [ ] Frontmatter `clicks: N` override accepted; the AST step plan is preserved regardless.
- [ ] Runtime reads `?step=N` from the URL, toggles class names (`slidev-vclick-current`, `-prior`, `-hidden`, etc.) on click targets.
- [ ] Click steps animate in via CSS transitions (`fade`, `up`, `scale`, composable modifiers).
- [ ] `next()` / `prev()` walk clicks before changing slides.
- [ ] Presenter view (future phase 10) and exports (future phases 12-13) will be able to introspect `slide.clickSteps` and `slide.totalClicks` without rendering â€” the data is in the slide module.
